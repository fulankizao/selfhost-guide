BookStack is a free, open-source wiki platform that organizes information into shelves, books, chapters, and pages. It's Docker-friendly, supports Markdown and WYSIWYG editing, and integrates with LDAP, SAML, and OAuth. As of 2026, BookStack has over 15,000 GitHub stars and is used by teams and individuals worldwide for documentation, knowledge bases, and personal wikis.

70| 71|

Why BookStack?

72|

Choosing the right wiki platform can be overwhelming. MediaWiki powers Wikipedia but is heavy — it requires a dedicated database server, PHP extensions, and frequent maintenance, making it overkill for personal or small-team use. DokuWiki is lightweight and file-based, but it lacks hierarchical content organization — all pages exist in a flat namespace, which becomes unmanageable as your knowledge base grows. Wiki.js offers a modern interface but relies on Node.js and a separate Git backend, adding complexity to both deployment and backup.

73|

BookStack strikes the right balance. Its Shelf → Book → Chapter → Page hierarchy mirrors how people naturally organize knowledge — like a real bookshelf. Shelves group related books (e.g., "DevOps" or "Recipes"), books contain chapters, and chapters hold individual pages. This structure scales gracefully from a few notes to hundreds of documents. It also runs on a single Docker Compose stack with PHP and MySQL — no extra runtimes or exotic dependencies. For official documentation, see the BookStack documentation.

Docker Compose Deployment

84| 
# docker-compose.yml
85|version: '3.8'
86|services:
87|  bookstack:
88|    image: lscr.io/linuxserver/bookstack:latest
89|    container_name: bookstack
90|    depends_on:
91|      - bookstack_db
92|    environment:
93|      - APP_URL=https://wiki.yourdomain.com
94|      - DB_HOST=bookstack_db
95|      - DB_USER=bookstack
96|      - DB_PASS=your_secure_password
97|      - DB_DATABASE=bookstack
98|    volumes:
99|      - ./data:/config
100|    ports:
101|      - "8080:80"
102|    restart: unless-stopped
103|
104|  bookstack_db:
105|    image: mysql:8.0
106|    container_name: bookstack_db
107|    environment:
108|      - MYSQL_ROOT_PASSWORD=root_password
109|      - MYSQL_DATABASE=bookstack
110|      - MYSQL_USER=bookstack
111|      - MYSQL_PASSWORD=your_secure_password
112|    volumes:
113|      - ./db:/var/lib/mysql
114|    restart: unless-stopped
115| 
docker compose up -d
116|# BookStack will be available at http://localhost:8080
117|# Default login: admin@admin.com / password
118|

For the latest image versions and available tags, check the LinuxServer BookStack image on Docker Hub.

121|

server {
123|    listen 80;
124|    server_name wiki.yourdomain.com;

Content Organization

149|

BookStack uses a simple hierarchy: Shelf → Book → Chapter → Page. Log in as admin and visit the Settings page to configure your instance name, registration policy, and authentication.

150|

Here is a practical walkthrough to get your wiki organized from day one:

151|
    152|
  1. Create a Shelf — From the homepage, click "Shelves" in the sidebar, then "Create a new Shelf". Give it a name (e.g., "DevOps Knowledge Base") and a short description. Shelves are your top-level categories and appear on the landing page. You can assign a cover image and set visibility permissions per shelf.
    2. 153|
  2. Add a Book — Open the shelf and click "Add Book". Name it something focused, like "Docker & Compose". Add a description that summarizes what the book covers. Within each book you can either add chapters (for sub-topics) or create pages directly.
    3. 154|
  3. Write Your First Page — Click "Create Page" inside a book or chapter. BookStack offers both a WYSIWYG editor and a Markdown editor — toggle between them with the editor mode button on the right. Write your content, add headings, insert code blocks using the <pre> button, and attach images by dragging them into the editor. Click "Save" when done.
    4. 155|
  4. Tag Pages for Discovery — While editing a page, open the Tags section at the bottom of the form. Add key-value tags like docker or tutorial. Tags are searchable and filterable, making it easy to find pages across different books.
    5. 156|
  5. Set Permissions — Each shelf, book, chapter, and page has its own permission settings. Click the "Permissions" tab and choose who can view, edit, or delete. You can restrict a shelf to specific roles (e.g., "Editors" or "Viewers") while keeping other shelves public. This is especially useful for team wikis where some content is sensitive.
    6. 157|
158|

For more on structuring content, refer to the official content structure documentation.

Authentication Integration

161|

BookStack supports multiple auth backends. For LDAP (useful for team setups):

162| 
# Settings → Authentication → LDAP
163|LDAP_SERVER=ldap://your-ldap-server:389
164|LDAP_BASE_DN=dc=example,dc=com
165|LDAP_DN=cn=admin,dc=example,dc=com
166|LDAP_PASS=your_ldap_password
167|LDAP_USER_FILTER=(&(objectClass=inetOrgPerson)(uid=${user}))
168|

BookStack also supports SAML2, OAuth with Google/GitHub/Slack, and social login. The LDAP configuration docs cover advanced options like group syncing and TLS.

Backup Automation

171|

Back up three things: the database, the uploads directory, and the .env file. See our backup strategies guide for a complete pipeline. A simple cron job:

172| 
#!/bin/bash
173|# /root/scripts/backup-bookstack.sh
174|docker exec bookstack_db mysqldump -u bookstack -p'password' bookstack > /backup/bookstack-$(date +%Y%m%d).sql
175|cp -r /path/to/bookstack/data/uploads /backup/uploads-$(date +%Y%m%d)
176|find /backup -name "*.sql" -mtime +30 -delete
177|

Store backups off-site using a tool like rclone to sync to S3, Backblaze B2, or a second server. The official backup guide has additional recommendations.

Theme Customization

180|

BookStack's default look is clean, but you can customize it to match your brand or personal preference. The easiest approach is installing a community theme from the BookStack GitHub repository or community sites like BookStack Themes — simply drop the theme folder into /config/www/themes/ and set APP_THEME=theme-name in your .env file.

181|

For custom CSS, add rules to the "Custom Stylesheet" field under Settings → Customization. You can override colors, fonts, spacing, and even the login page background. To replace the logo, upload a new image via Settings → Customization → App Logo — BookStack supports SVG, PNG, and WebP formats. All customizations survive Docker container updates as long as they are stored in the persistent /config volume. See the visual customization docs for detailed instructions.

Performance Optimization

184|
    185|
  • Cache: Enable Redis caching for faster page loads — add a redis service to your Docker Compose and set CACHE_DRIVER=redis

  • Uploads: Store uploads on a dedicated volume or S3-compatible storage to keep the app container lean

  • App Debug: Set APP_DEBUG=false in production to disable verbose error logging and speed up response times
    • 188|
  • Cron: Set up a cron job for BookStack's scheduled tasks (search indexing, cleanup) using the provided artisan schedule:run command

Troubleshooting Common Issues

192|
    193|
  • Port conflict — If port 8080 is already in use (e.g., by another Docker container), change the host port in docker-compose.yml to something else like "8081:80", then update your Nginx proxy_pass accordingly.
    • 194|
  • Database connection failure — BookStack cannot start if the MySQL container isn't ready. Add depends_on conditions or a health check. Check logs with docker logs bookstack_db and ensure DB_HOST matches the service name. If using an external database, verify firewall rules allow traffic.
    • 195|
  • Permission errors — The /config volume must be writable by the container's www-data user. If you see "Failed to write" errors, run chown -R 1000:1000 ./data on the host (the LinuxServer image uses UID 1000).
    • 196|
  • 404 after migration — If you moved BookStack to a new server or changed the APP_URL, cached routes may break. Run docker exec bookstack php artisan cache:clear and restart the container. Also check that your Nginx proxy_pass URL matches the internal port.
    • 197|
198|

For more help, visit the BookStack GitHub Issues or the community Discord server.

FAQ

201|
    202|
  • Can I use SQLite instead of MySQL? Yes, BookStack supports SQLite for small single-user setups — just set DB_CONNECTION=sqlite and remove the MySQL service from your compose file. However, SQLite does not support concurrent writes well, so MySQL/MariaDB is recommended for production or any multi-user scenario.
    • 203|
  • How do I update BookStack? Pull the latest Docker image with docker compose pull && docker compose up -d, then run the migration script via docker exec bookstack php artisan migrate to apply any database schema changes. Always back up your database before upgrading.
    • 204|
  • Can BookStack integrate with existing auth systems? Yes — LDAP, SAML2, OAuth (Google, GitHub, Slack), and social login. You can also configure automatic user registration and role mapping, so new users from your IdP are added to the right groups without manual admin work.
    • 205|
  • Is BookStack suitable for team use? Yes, with role-based permissions (Viewer, Editor, Admin), per-shelf ACLs, full-text search across all content, and an audit log that tracks who changed what and when. It is built for teams of 5 to 500.
    • 206|
  • How do I back up BookStack? Back up the MySQL database (mysqldump), the uploads directory, and the .env file. See our backup strategies guide for a complete automated pipeline with off-site storage.
    • 207|
