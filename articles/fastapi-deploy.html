FastAPI Project: From Development to Deployment

2026-06-15 · 15 min read · views
← Back to Home

FastAPI is one of the fastest Web frameworks in the Python ecosystem, featuring auto-generated API docs, Pydantic validation, and async support. This guide walks through the entire process from project scaffolding to production deployment with Nginx reverse proxy + Docker.

According to the FastAPI GitHub repository, the framework has surpassed 80,000 stars and is one of the fastest-growing Python projects on the platform. The 2025 Stack Overflow Developer Survey ranked FastAPI as the second most loved web framework overall, and its adoption in production environments has grown by over 50% year-over-year among Python developers — making it a critical skill for modern backend engineering.

1. Project Structure

A well-organized FastAPI project structure:

my-api/
├── app/
│   ├── __init__.py
│   ├── main.py           # Application entry point
│   ├── config.py         # Configuration management
│   ├── database.py       # Database connection
│   ├── models/           # SQLAlchemy models
│   ├── schemas/          # Pydantic validation
│   └── routers/          # API routes
├── requirements.txt
├── Dockerfile
├── docker-compose.yml
└── .env

2. Quick Start

# Installation
pip install fastapi uvicorn sqlalchemy pydantic

# Minimal main.py
from fastapi import FastAPI
app = FastAPI(title="My API")

@app.get("/")
def root():
    return {"message": "Hello World"}

@app.get("/health")
def health():
    return {"status": "ok"}
# Start dev server
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
# Visit http://localhost:8000/docs to see Swagger docs

For detailed API development patterns, refer to the official FastAPI documentation, which includes comprehensive guides on path operations, dependency injection, and security utilities.

3. Database Integration

# app/database.py
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, DeclarativeBase

DATABASE_URL = "sqlite:///./data.db"
engine = create_engine(DATABASE_URL)
SessionLocal = sessionmaker(bind=engine)

class Base(DeclarativeBase):
    pass

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

4. Managing with systemd

Similar to Hermes Agent's service management approach, set up FastAPI as a systemd service:

sudo tee /etc/systemd/system/fastapi-app.service > /dev/null << 'EOF'
[Unit]
Description=FastAPI App
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/my-api
ExecStart=/opt/my-api/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl enable --now fastapi-app

5. Nginx Reverse Proxy

# /etc/nginx/sites-available/my-api
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /static {
        alias /opt/my-api/static;
        expires 7d;
    }
}

# Enable the site
sudo ln -s /etc/nginx/sites-available/my-api /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

6. Docker Deployment

# Dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
# docker-compose.yml
version: '3.8'
services:
  api:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data
    environment:
      - DATABASE_URL=sqlite:///data/data.db
    restart: unless-stopped

FAQ

What is the difference between FastAPI and Flask?

Flask is a synchronous micro-framework with manual extension configuration, while FastAPI is ASGI-native with automatic OpenAPI docs, built-in Pydantic validation, and async support out of the box. FastAPI generally offers better performance (on par with Node.js and Go) and a more modern developer experience with automatic request/response validation.

Do I need to use async with FastAPI?

No. FastAPI supports both sync and async route handlers. Use async for I/O-bound operations (database queries, HTTP calls, file I/O) to get the performance benefits. For CPU-bound tasks, stick with sync or use background tasks. FastAPI automatically handles the threading correctly for sync endpoints.

How do I add authentication to my FastAPI app?

FastAPI provides built-in support for OAuth2, JWT tokens, HTTP Basic auth, and API keys through its dependency injection system. The official FastAPI documentation includes a complete OAuth2 with JWT tutorial. For self-hosted deployments, you can also put authentication behind an Nginx reverse proxy or Cloudflare Access.

What ASGI server should I use for production?

For production, use Gunicorn with Uvicorn workers (pip install gunicorn uvicorn): gunicorn -w 4 -k uvicorn.workers.UvicornWorker app.main:app. This combines Gunicorn's process management with Uvicorn's ASGI performance. For simpler deployments, Uvicorn alone with --workers 4 is also acceptable.

How do I handle CORS in FastAPI?

FastAPI has a built-in CORSMiddleware. Simply add from fastapi.middleware.cors import CORSMiddleware and configure allowed origins, methods, and headers. This is essential when your frontend runs on a different domain than your API.
